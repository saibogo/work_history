  1. Назначение системы

    Данная система разрабатывалась для нужд технической службы Екатеринбургского отделения компании "Малахит" и предназначена для хранения и сбора информации следующего характера:

    1. Текущем составе технической службы, статусе сотрудников и их графиках
    2. Хранения общей техничсекой информации об осьслуживаемых предприятиях
    3. Хранения информации об эксплуатируемом оборудовании, историй его ремонта и перемещений
    4. Хранения вспомогательной информации об оборудовании, как то деталировки и инструкции
    5. Хранения информации о зарегистрированных и выполненных заявках
    6. Хранения информации о выполненных работах и составе бригад, их выполнявших
    7. Хранения информации об установленных приборах учета, истории их показаний(в отдельных случаях и профили мощности). Также для некоторых пользователей доступен расчет текущего потребления согласно имеющимся схемам расчета по приборам учета

  2. Возможные способы использования системы

    Существует два основных способа использовать портал:

    1. Компьютер, подключенный к внутренней сети компании(т.е. обладающий необходимым IP адресом). Важно помнить что поддерживаются не все браузеры(см. Пункт Главное меню)
    2. Любое устройство с установленным Telegram. Важно помнить, что данный способ не обеспечивает Полной функциональности браузерного варианта

  3. Получение доступа к системе

    Система не предусматривает автоматическое создание нового пользователя. Более того следует понимать, что с точки зрения програмного комплекса, доступ через браузер и доступ через Telegram - это доступ РАЗНЫХ пользователей. Поэтому в зависимости от того, как именно вы хотите использовать портал первоначальный доступ можно получить следующими способами:

    1. Придумать уникальный логин и пароль в соотвествии с требованиями соотвествующего раздела. Далее обратиться к администратору системы сообщив ему и первое и второе с просьбой зарегистрировать новую учетную запись для доступа через браузер
    2. Подключить в Telegram бот MalachiteWorksBot. Далее необходимо попросить администратора системы подключить новую учетную запись. Системный администратор попросит Вас отправить некое сообщение данному боту после чего сможет увидеть ваш идентификатор и внести его в базу данных
    3. Важно помнить, что часть функций портала доступна БЕЗ регистрации(например просмотр текущего графика сотрудников)

  5. Главное меню веб-версии портала

    Главное меню содержит нижеперечисленные пункты. Важно помнить, что в данном ПО реализована система разделения прав пользователей, что означает доступность либо недоступность каждого пункта меню для разного типа пользователей. Также следует помнить, что даже если конкретный пункт меню вам доступен, это не означает, что будут доступны все пунты вложенного меню либо все вложенные действия

    1. Операции с предприятиями

      Данный пункт меню позволяет просматривать информацию по зарегистрированным в данное время предприятиям, а также при необходимости добавлять новые либо менять статус уже существующих. При нажатии на данный пункт будет открыто меню для работы с предприятиями, содержащее следующие подпункты:

        Действия с предприятиями:

      1. Все предприятия - список всех предприятий, зарегистрированных в системе
      2. Только действующие - аналогично предыдущему, но предприятия либо функционируют, либо находятся на реконструкции
      3. Добавить предприятие - только для администратора. Переход к меню регистрации нового предприятия
      4. TOP-10 по количеству ремонтов - небольшая статистика по количеству ремонтов на предприятиях

        5. Важно уточнить - с точки зрения программы существует три вида предприятия:

        • Независимое, т.е. состоящее ровно из одного обьекта
        • Комплекс - то есть включающее в себя два и более предприятий
        • В составе комплекса - то есть являющееся частью комплекса
    2. Операции с оборудованием

      Данный пункт меню позволяет просматривать информацию по всем единицам зарегистрированного оборудования, а также работать с деталировками и инструкиями для оного

        Действия с оборудованием

      1. Все зарегистрированное оборудование - Все единицы или группы оборудования, когда либо регистрировавшиеся в системе
      2. Поиск по ID - поиск оборудования по уникальному идентификатору
      3. TOP-10 по ремонтам - Краткая статистика по наиболее часто ремонтируемому оборудованию
      4. Работа с деталировками - переход в подменю для работы с типами оборудования, деталировками и инструкциями

    3. Операции с ремонтами

      Данный пункт меню содержит всего два подпункта: Все зарегистрированные работы и Поиск работы по ID. Собственно первый пункт переключает вас на таблицу со всеми, когда либо зафиксированными в журнале работами. Каждая выполненная работа имеет свой уникальный идентификатор - ID. Соотвественно если вы знаете данный ID, то можете переключиться не на весь перечень, а на строго определенную запись


    4. Операции с сотрудниками

      Данный пункт меню позволяет просматривать всю необходимую служебную информацию по сотрудникам технической службы а также добавлять новых

        Действия с сотрудниками

      1. Все зарегистрированные сотрудники - список всех когда либо зарегистрированных сотрудников техслужбы
      2. Только работающие - то же, что предыдущий пункт но без учета уволенных
      3. TOP-10 по зарегистрированным работам - небольшая статистика по произведенным работам
      4. Добавить сотрудника - только для администратора системы. Позволяет зарегистрирвоать нового сотрудника
      5. Сотрудники на смене - открывает меню с текущими и архивными графиками сотрудников

    5. Баг-трекер системы

      Данный пункт главного меню предназначен для хранения информации обо всех выявленных пользователями проблемах при работе системы

        Некорректное поведение системы

      1. Отобразить все - Список всех выявленных проблем, независимо от их статуса
      2. Отобразить незакрытые - Список всех нерешенных на данный момент проблем
      3. Зарегистрировать проблему - Авторизоваться и зарегистрировать новую проблему

    6. Заявки и заказчики

      Данный пункт предназначен для создания, хранения и отображения всей доступной информации как о самих заявках, так и о тех, кто имеет право работать с заявками. Далее приведено краткое описание подпунктов

        Действия с заявками

      1. Все заказчики - только для администратора. Позволяет корректировать данные заказчиков
      2. Добавить заказчика - только для администратора. Позволяет создать нового заказчика
      3. Все зарегистрированные заявки - список всех когда либо созданных заявок
      4. Только незакрытые заявки - список заявок по всем предприятиям находящихся в работе
      5. Поиск по ID - поиск в системе заявки с уникальным идентификатором
      6. Зарегистрировать заявку - создать новую заявку от текущего пользователя. Внимание, требуется авторизация
      7. Только мои заявки - список незакрытых заявок, принадлежащих данному пользователю, либо всех незакрытых если пользователь является администратором системы

    7. Изменить тему оформления

      Данный пункт предназначен для изменения визуальной составляющей отображаемой страницы. Всего на данный момент доступны три темы оформления. Последовательное нажатие на данный пункт меню приведет к смене данной составляющей. Доступно только после авторизации.


    8. Изменения в системе

      Данный пункт открывает таблицу, отображающую все зарегистрированные изменения в системе с указанием версии ПО и даты внедрения изменений


    9. Вспомогательные сервисы

      Данный пункт переводит вас в подменю, в котором собрано все, что не имеет непосредственного отношения к изначальным целям системы.

        Доступные сервисы

      1. Планируемые отключения электроэнергии - Открывает пользователю таблицу с планируемыми отключениями электроэнергии по данным компаний Россети-Урал и Екатеринбургэнергосбыт. Отбор ведется по исходным шаблонам, указанным в нижней части страницы. Внимание - при переходе возникнет пауза, связанная с загрузкой и анализом исходных данных!
      2. Приборы учета - Перечень всех приборов учета, фигурирующих в системе. Внимание - приборы учета не разнесены по предприятиям!

    10. Инструкция

      Данный пункт переводит вас в раздел Инструкция. Данный раздел подразделен соотвественно на инструкции для Рядового Пользователя, Системного Администратора и Разработчика


    11. Выйти из системы

      Данный пункт закрывает текущую авторизацию пользователя. Крайне рекомендуется выполнять эту операцию на компьютерах, к которым есть доступ нескольких сотрудников. Принудительный выход из системы осуществляется через шесть часов отсутствия активности пользователя в системе


  6. Структура нижней панели навигации

    Данная панель включается опционально для разных страниц и она всегда расположена в нижней части страницы

    1. В предыдущее меню - данный пункт переводит вас на ту страницу, которая УСТАНОВЛЕНА для текущей как изначальная. Внимание - данный пункт не равноценен нажатию кнопки браузера НАЗАД.

    2. Главное меню - переключает вкладку на главное меню программы

    3. Обратная связь - открывает установленный почтовый клиент, либо вкладку с почтовой программой по умолчанию, для создания сообщения разработчику программы

    4. Частые вопросы - открывает вкладку с перечнем простых вопросов и ответов по системе
